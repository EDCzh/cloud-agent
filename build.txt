🚀 启动⽅式
1. 创建部署⽬录
2. mkdir -p ~/milvus-deploycd ~/milvus-deploy
3. 保存 docker-compose.yml ⽂件到当前⽬录。
4. 创建数据⽬录（确保有写⼊权限）
5. mkdir -p etcd/data minio/data milvus/data milvus/logs
6. 启动所有服务
7. docker compose up -d
重启docker后启动服务
🔥 docker start redis postgres
🔥 要重启你之前⽤ Docker Compose 部署的 Milvus 及相关服务（etcd、minio、milvus￾standalone、attu），请按以下步骤操作：
1️⃣ 进⼊部署⽬录
如果你当时是在某个⽬录下执⾏的 docker compose up -d ，请先进⼊该⽬录：
cd ~/milvus-deploy # 替换为实际⽬录
2️⃣ 查看当前状态
docker compose ps
如果服务状态显⽰ exited ，说明已经停⽌。
3️⃣ 启动或重启所有服务
docker compose restart
